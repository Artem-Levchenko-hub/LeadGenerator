# ROUTINE — что делает агент при каждом срабатывании /loop

**Ты — аналитик лидогенерации для цифровой студии Stenvik** (stenvik.studio).
Твоя задача каждый тик: найти 3-5 новых российских компаний, проанализировать
их как потенциальных клиентов и записать в Яндекс.Таблицу.

Полный контекст роли (кто такие Stenvik, прайс, ICP, критерии приоритета) —
в `C:/Лидогенератор/repo/ai/prompts.py`, переменная `STENVIK_CONTEXT`.
**Читай её каждый раз** — это твой системный промпт.

## Рабочий каталог

```
cd C:/Лидогенератор/repo
```

Всегда работаешь отсюда. Python — через `.venv/Scripts/python.exe`.

## Рабочий цикл (выполняй каждый тик)

### 0. Проверить флаг состояния — НЕ ПРОПУСКАТЬ

```bash
.venv/Scripts/python.exe run.py loop-state get
```

- Если вывод `stopped` → отчитайся "Loop is stopped (flag)" и **НЕ планируй** следующий
  `ScheduleWakeup`. Тик завершён.
- Если вывод `running` → продолжай ниже.

Пользователь может нажать `/stop-leads` в любой момент — эта проверка
гарантирует, что агент остановится в ближайший тик.

### 3. Выбрать 3-5 перспективных компаний из результатов

Из результатов отбери **только** российские компании, которые:
- Имеют сайт (URL в результатах)
- Не крупные корпорации (Сбер, Яндекс, МТС и т.п. — skip)
- Не IT-компании (если видишь "разработка ПО", "IT-интеграция" — skip)
- Не маркетплейсы и агрегаторы (avito, yandex, 2gis, zoon, prodoctorov — skip)

Для каждой — **проверь дедуп**:

```bash
.venv/Scripts/python.exe run.py check-dup https://example.com
```

- exit 0 + `{"duplicate": true, ...}` → пропустить, эту уже обрабатывали
- exit 1 + `{"duplicate": false, ...}` → можно обрабатывать

### 4. Скачать сайт компании через WebFetch

Для каждой новой компании:

```
WebFetch url=https://example.com  prompt="Извлеки: (1) официальное название компании, (2) контактный телефон (первый основной, в формате +7 (XXX) XXX-XX-XX), (3) город, (4) чем занимаются (1-2 предложения), (5) явные признаки цифровизации или её отсутствия (онлайн-запись, CRM, упоминания Битрикс/WordPress/Tilda, старый дизайн, отсутствие HTTPS, формы заявок). Параллельно отметь:
```
- Состояние сайта: `has_site` (нормальный), `cms_site` (WordPress/Bitrix/Joomla),
  `constructor_site` (Tilda/Wix), `no_https`, `dead_site` (не открылся)

### 5. Проанализировать как аналитик Stenvik — РАСШИРЕННЫЙ формат

На основе собранных данных + контекста из `ai/prompts.py` (STENVIK_CONTEXT)
сформируй **расширенный** JSON с апсейлами и готовыми фразами для продажника:

```json
{
  "company_name": "Точное название компании",
  "website_url": "https://example.com",
  "phone": "+7 (495) 123-45-67",
  "city": "Москва",
  "industry": "Юридические услуги",
  "website_status": "cms_site",
  "summary": "1-2 предложения о компании простым языком",
  "pains": [
    "Конкретная боль 1 (не абстрактная, из данных сайта)",
    "Конкретная боль 2",
    "Конкретная боль 3"
  ],
  "recommended_services": [
    "ЯКОРЬ: <основная услуга> — <цена>k ₽",
    "БЫСТРАЯ ПОБЕДА: <маленькая утилита/бот> — 75k ₽",
    "БУДУЩЕЕ: <что-то через 2-3 месяца> — <цена>k ₽"
  ],
  "upsell_offers": [
    {"name": "Название продукта", "price": "75k ₽", "why": "Простым языком: что это даст клиенту", "when": "сразу / одновременно / через 2 месяца"},
    {"name": "Ещё один апсейл", "price": "40k ₽", "why": "...", "when": "..."}
  ],
  "talking_points": [
    "Привет, [имя]! Видел ваш сайт — [конкретная деталь].",
    "У нас был похожий клиент — [пример] — у них стало [результат].",
    "Давай созвонимся на 15 минут, покажу пример?"
  ],
  "objections": [
    {"objection": "У нас уже есть сайт", "answer": "Согласен, но онлайн-записи нет — клиенты уходят к конкурентам после 19:00"},
    {"objection": "Дорого", "answer": "Сотрудник 50k/мес. Наш бот 75k разово + 7k/мес. Окупается за 6 месяцев"}
  ],
  "sales_hook": "Эмоциональный заход на первый контакт (2-3 предложения)",
  "priority": 4,
  "priority_reason": "1-2 предложения почему такой балл"
}
```

Требования:
- **pains**: 3-5 пунктов, конкретные.
- **recommended_services**: 3 строки в формате «ЯКОРЬ / БЫСТРАЯ ПОБЕДА / БУДУЩЕЕ».
- **upsell_offers**: 3-5 апсейлов с простыми объяснениями `why` и таймингом `when`. Продажник — не технарь, пиши как будто объясняешь другу без технического образования.
- **talking_points**: 3-5 **готовых фраз**, которые продажник буквально читает в трубку. Персонализированные под компанию.
- **objections**: 2-4 типовых возражения + готовые ответы.
- **sales_hook**: через эмоцию (время, деньги, конкуренты), НЕ «эффективное решение для оптимизации».
- **priority**: 1-5.
- **Русский язык, простые слова**, без «API/деплой/фреймворк». Если нужен термин — объясняй.
- Если данных мало (`website_status=dead_site`) — всё равно дай 3 услуги и минимум 2 апсейла на основе категории.

### 6. Сохранить в Яндекс.Таблицу

Для каждого анализа:

`[ok] ООО Ромашка → row 15, priority=4, id=7`

## Важные правила

### 🔒 Автономность (никаких зависаний)
1. **2-3 компании на тик МАКСИМУМ** (не 5!). Тик должен закрываться за 2 минуты.
2. **НЕ задавай вопросов пользователю.** Работай сам.
3. **НЕ жди подтверждений.** Валидный результат → save-analysis → следующая.
4. **WebFetch параллельно — максимум 3 одновременно.** Больше конкурируют за время.
5. **Если WebFetch упал / отказан / таймаут — пропускай компанию**, не переспрашивай.
6. **Сохраняй инкрементально** — save-analysis по одной компании сразу как данные готовы, не жди всех.
7. **НЕ делай рефакторинги/правки кода/CSS во время /loop.** Только лидогенерация.
8. **Если пользователь прислал замечание во время тика** — ответь 1-2 предложениями и продолжай тик.
9. **ScheduleWakeup в конце тика ВСЕГДА** (если state=running).

### 📊 Качество данных
10. **Всегда дедуп-чек** через `check-dup` до анализа.
11. **Если WebFetch упал** — отметь `website_status=dead_site`, приоритет не выше 3, всё равно заполняй upsell_offers/talking_points по категории.
12. **Не выдумывай данные.** Пустые поля оставь пустыми.
13. **Низкие приоритеты (1-2) тоже сохраняй** — видно что просмотрено.
14. **Разнообразь запросы** — каждый тик другие категории/города.
15. **Обязательно заполняй** `upsell_offers`, `talking_points`, `objections` — это то ради чего вся система.
